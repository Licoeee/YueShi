# 欢迎界面重设计说明

## 1. 目标

将现有欢迎页从“烘焙摄影 + Lottie 字标”的旧方案，调整为可直接落地的品牌入场页。新方案需要同时满足品牌辨识度、手机首屏构图稳定性，以及后续角色分流与转场动效的实现边界。

## 2. 已确认的核心决策

- 欢迎页为小程序**每次启动的固定入口**。
- 欢迎页点击“进入主页”后，再执行角色识别与分流。
- 欢迎页采用 `B` 方向构图：品牌字在上，蛋糕主视觉居中，CTA 按钮固定在下。
- 当前确认的是**静态视觉基线**，后续再在此基线上补按钮入场动画和主页 reveal 转场。

## 3. 首屏视觉基线

### 3.1 背景

- 使用暖粉到日落橙的径向渐变背景。
- 背景中心偏亮，边缘向桃橙色过渡，保持轻微逆光氛围。
- 背景只负责烘托，不再使用烘焙摄影图。

### 3.2 标题区

- 标题文案：`悦时`
- 副标题文案：`遇见你的专属甜蜜时刻`
- 标题位置：位于页面上部安全区下方，尺寸明显放大，占据用户已确认的红框区域。
- 字体要求：
  - 标题需呈现圆润、厚重、甜点品牌感。
  - 标题保持半通透感，不能做成实体描边字。
  - “时”字的内部细线层次通过阴影呈现，不使用外描边。
  - 副标题需要更高字重与更疏朗的字距，避免拥挤。

### 3.3 蛋糕主视觉

- 蛋糕位于欢迎页视觉中轴。
- 蛋糕风格为“贴近真实结构的卡通鲜奶蛋糕”，而不是纯几何块面。
- 顶部固定为三根蜡烛。
- 奶油层为白色或微粉白色。
- 蛋糕胚改为淡粉色。
- 奶油上的装饰为零散分布的圆形彩色糖果。
- 草莓装饰已废弃，不再进入实现方案。

### 3.4 CTA 按钮

- 文案固定为：`进入主页`
- 按钮位于底部安全区上方，使用 TDesign 主按钮视觉语义。
- 常规态保持粉橙渐变、白色文字、轻微高光边缘和外发光阴影。

## 4. 动效分镜基线

### 4.1 入场顺序

1. 欢迎页背景与标题先稳定出现。
2. 中央蛋糕进入轻微漂浮态。
3. “进入主页”按钮从中心向两侧展开，并逐步由虚化过渡为实体。

### 4.2 点击后的转场

- 点击按钮后，转场原点固定在**屏幕中心**，不是按钮中心。
- 使用圆形 reveal 逐步显示主页内容。
- reveal 过程需要加入柔和 easing，避免突兀切页。
- reveal 边缘可带少量粒子拖尾，增强“展开”的生命力。
- 主页内容需要在 reveal 扩张过程中逐步可见，直到完全铺满屏幕。

## 5. 角色分流与状态边界

- 欢迎页点击后开始或继续静默获取 `OpenID`。
- 角色识别结果在 reveal 结束前准备好，结束后立即进入对应页面。
- 管理员保留“欢迎回来，最高指挥官大帅！”提示语。
- 商家保留“生意兴隆，[店名]的老板！”提示语。
- 顾客无额外欢迎弹层，直接进入顾客端首页。

## 6. 实现约束

- 欢迎页不再依赖 Lottie 作为主视觉方案。
- UI 组件仍以 TDesign 为基础，按钮必须保持 TDesign 语义与接入方式。
- 核心视觉蛋糕可使用页面结构或内嵌 SVG 实现，但必须保证在微信开发者工具 `2.01.2510280` 中可稳定渲染。
- 长背景继续遵守“固定背景层 + 内容层”结构。

## 7. 验收标准

- 用户每次打开小程序均先看到欢迎页。
- 标题、蛋糕、按钮的手机首屏比例与当前确认稿一致。
- 标题不再出现硬描边感；蛋糕不再出现草莓装饰。
- 按钮出现动画和点击 reveal 转场都具备明确的缓动与层次，不是机械切换。
- reveal 结束后进入正确的角色主页。
