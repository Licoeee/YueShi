# 欢迎页 2.1 设计说明

## 1. 目标

交付 TODO 2.1 的欢迎页视觉与动效基线，让小程序每次启动都先进入品牌欢迎页，并在点击 CTA 后以圆形 reveal 转场进入当前主页占位页，为后续 2.2 OpenID 静默获取和 2.3 角色分流预留清晰接入点。

## 2. 已确认方案

### 2.1 路由与页面职责

- 采用独立页面方案：新增 `pages/welcome/welcome` 作为小程序首屏入口。
- 当前 2.1 不接入 OpenID、角色识别、角色欢迎语和三端分流逻辑。
- 点击 `进入主页` 后，欢迎页执行完整 reveal 动效，再统一跳转到当前顾客首页占位页 `pages/index/index`。

### 2.2 页面结构

- 欢迎页采用“固定背景层 + 内容层”结构，规避微信小程序长页面渐变断层问题。
- 页面内容按纵向中轴排布：标题区在上、蛋糕主视觉居中、CTA 位于底部安全区上方。
- 页面状态保持极简，仅保留欢迎页可见性、按钮就绪态、reveal 进行态和粒子数据。

## 3. 视觉设计

### 3.1 背景

- 使用暖粉到日落橙的径向渐变背景：中心偏亮，边缘向桃橙过渡。
- 渐变只挂在独立固定背景层，不直接挂到全局 `page`。

### 3.2 标题区

- 主标题文案：`悦时`
- 副标题文案：`遇见你的专属甜蜜时刻`
- 主标题使用圆润厚重字体栈 `YouYuan, FZYaoti, "PingFang SC", "Microsoft YaHei", sans-serif`。
- 主标题颜色使用半通透暖玫瑰色，依赖柔和阴影建立层次，禁止硬描边。
- 副标题使用暖棕色、更高字重和更疏朗字距，保证首屏可读性。

### 3.3 蛋糕主视觉

- 蛋糕主视觉不使用外部图片，直接由 WXML 结构和 WXSS 造型组成。
- 蛋糕包含三根蜡烛、火苗、白色或微粉白奶油层、淡粉色蛋糕胚和零散分布的圆形彩色糖果。
- 禁止引入草莓装饰或字体图标依赖。

### 3.4 CTA 按钮

- 使用 TDesign `t-button`，按钮文案固定为 `进入主页`。
- CTA 放置在底部安全区上方，维持品牌渐变、内高光边缘和暖色外发光阴影。
- 按钮首次进入时，必须从中心向四周展开，并伴随透明度提升与轻微虚化收敛。

## 4. 动效设计

### 4.1 静态入场

1. 背景和标题先稳定出现。
2. 中部蛋糕进入轻微漂浮状态。
3. CTA 从中心向四周展开，完成后才允许点击。

### 4.2 常驻微动效

- 蛋糕整体使用轻微上下漂浮动画。
- 三根蜡烛火苗使用呼吸动画。
- 动画以 CSS 状态切换驱动，不引入 Lottie 或复杂逐帧逻辑。

### 4.3 点击转场

- 用户点击 `进入主页` 后，立即锁定按钮，禁止重复触发。
- 页面叠加一个以屏幕中心为原点的圆形 reveal 遮罩层。
- reveal 需具备柔和 easing，并附带少量粒子拖尾增强层次感。
- reveal 完成后再跳转到 `pages/index/index`。

## 5. 实现边界

- 2.1 只实现欢迎页视觉、动效和占位跳转。
- 2.2 再接入 OpenID 静默获取。
- 2.3 再接入管理员、商家、顾客分流和对应欢迎消息。
- 欢迎页核心视觉优先使用原生 WXML/WXSS/TS 状态驱动实现，不依赖 Lottie。

## 6. 验收标准

- 小程序启动后首屏进入欢迎页，而不是直接进入 `pages/index/index`。
- 标题具备半通透、柔和阴影和圆润厚重字体气质，无硬描边。
- 蛋糕主视觉轮廓需接近参考稿的“两层鲜奶蛋糕”结构，能直观看出是生日蛋糕，而不是抽象几何堆叠。
- CTA 视觉只能保留单层主体，不得出现按钮底座和按钮本体双重叠色。
- CTA 从中心向四周展开并逐渐实体化后才进入可点击状态。
- 点击 CTA 后，以屏幕中心为原点的展开区域需要实时露出主页首屏预览内容，而不是仅显示单独的扩散色块；展开结束后再跳转到 `pages/index/index``。
- 页面在微信开发者工具 `2.01.2510280` 中可稳定预览。
