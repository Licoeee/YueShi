# 第三阶段顾客端首页与详情购买链路设计说明

## 1. 目标

交付第三阶段顾客端的首批核心浏览购买链路，让用户可以在顾客首页浏览蛋糕瀑布流、点击卡片查看半屏详情、在详情区域内上滑进入独立详情页，并在独立详情页完成规格选择、加入购物车和立即购买。

## 2. 已确认范围

### 2.1 首页范围

- 顾客首页当前阶段只展示蛋糕商品。
- 首页使用双列瀑布流承载商品卡，支持图片懒加载、关键词或 ID 搜索、分类筛选和价格/销量排序。
- 商品卡片右下角不再放置快捷加购图标。

### 2.2 详情查看链路

- 用户点击商品卡后，先进入自屏幕底部上弹的半屏详情层。
- 半屏详情层只承担展示职责，不承载规格选择、加入购物车或立即购买操作。
- 半屏详情层允许查看蛋糕详情图，并保留被压暗的首页背景。
- 用户在半屏详情区域内继续上滑后，进入独立详情页。

### 2.3 独立详情页职责

- 独立详情页承接完整商品详情、规格选择和购买行为。
- 规格至少包含层数、尺寸或尺寸搭配、奶油类型。
- 单层蛋糕直接选择尺寸；多层蛋糕先选层数，再选对应尺寸搭配。
- 独立详情页底部固定四个操作位：首页、购物车、加入购物车、立即购买。

### 2.4 购物车语义

- 点击加入购物车：商品写入购物车，不跳转页面。
- 点击立即购买：商品写入购物车，并将该商品设置为已勾选状态，然后跳转购物车页。
- 当前阶段购物车先使用本地缓存持久化。
- 购物车云端同步不纳入本次实现，但必须作为第三阶段后续任务承接。

## 3. 页面与组件职责

### 3.1 页面结构

- 保留 `pages/customer/home/home` 和 `pages/customer/cart/cart` 作为顾客端 role scene 页面外壳，不破坏第二阶段已经稳定的场景切换与底部导航机制。
- 新增 `pages/customer/product-detail/product-detail` 作为独立详情页。

### 3.2 组件拆分

- `role-page-scene` 继续作为顾客端场景宿主，但对顾客首页和顾客购物车路径改为挂载真实业务场景组件。
- 新增 `customer-home-scene` 组件，负责首页商品列表、搜索筛选、卡片点击和半屏详情层状态。
- 新增 `customer-product-sheet` 组件，负责半屏详情展示态。
- 新增 `customer-product-detail` 组件，负责独立详情页的商品图、规格选择和底部操作区。
- 新增 `customer-cart-scene` 组件，负责购物车最小真实展示态，为立即购买跳转提供承接页。

## 4. 数据模型

### 4.1 商品模型

- 继续保留现有 `ProductSummary` 作为列表摘要类型。
- 扩展蛋糕详情结构，补充：
  - 详情图列表
  - 层数选项
  - 单层尺寸选项
  - 多层尺寸搭配选项
  - 奶油类型选项
- 当前阶段数据来源为本地 mock 数据，不直接接云数据库。

### 4.2 购物车模型

- 为购物车新增独立类型，记录：
  - 商品标识与标题
  - 封面图
  - 数量
  - 已选层数
  - 已选尺寸或尺寸搭配
  - 已选奶油类型
  - 单价与总价
  - 是否勾选
  - 入车方式（普通加购或立即购买）

## 5. 关键交互流

### 5.1 首页到半屏详情

1. 首页加载本地蛋糕列表并完成瀑布流排布。
2. 用户点击商品卡。
3. 页面叠加暖灰或暖棕调暗色遮罩。
4. 半屏详情层从底部上弹。

### 5.2 半屏详情到独立详情页

1. 用户在半屏详情区域内继续上滑。
2. 当前商品上下文透传至独立详情页。
3. 独立详情页复用同一商品详情内容，不重新切换成另一套信息结构。

### 5.3 独立详情页购买动作

1. 用户选择层数。
2. 用户根据层数选择尺寸或尺寸搭配。
3. 用户选择奶油类型。
4. 点击加入购物车时，写入本地购物车，停留在当前页。
5. 点击立即购买时，写入本地购物车，将当前商品标记为已勾选，并跳转到购物车页。

## 6. 视觉与动效

### 6.1 首页

- 首页延续暖粉、桃橙、奶油白主色调。
- 瀑布流卡片维持柔和高光边缘和暖色阴影，不使用硬黑描边。

### 6.2 半屏详情层

- 半屏详情层默认不占满屏幕。
- 顶部保留被压暗的原页面露出区，避免直接切断首页上下文。
- 半屏态不显示底部购买操作区，只承担展示和上滑引导。

### 6.3 独立详情页

- 独立详情页延续半屏态商品内容，但布局允许纵向展开。
- 底部操作区固定悬浮，图标入口和实体按钮需要主次清晰。
- 详情图支持点击放大查看。

### 6.4 动效

- 半屏详情层上弹使用 `Lift Sheet` 动效，时长建议 280ms 到 360ms。
- 半屏详情页进入独立详情页使用 `Detail Expand` 动效，触发条件为详情区域内上滑，强调拖拽连续感。

## 7. 实现边界与阶段排期

- 本次实现以 `3.1 首页瀑布流` + `3.2 商品详情页与购买链路` 为核心。
- 为了支撑立即购买跳购物车，本次允许补齐 `3.3 购物车本地持久化` 的最小基础能力。
- 购物车云端同步不在本次落地，但数据结构和存储接口要为后续同步留出入口。
