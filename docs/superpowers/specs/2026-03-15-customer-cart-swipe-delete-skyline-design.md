# 顾客购物车滑删 Skyline 兼容设计

## Goal

修复顾客端购物车在手机扫码真机环境下左滑删除无响应的问题，同时保持现有的商品展示、勾选、全选、金额汇总与去结算能力不变。

## Root Cause

- 当前购物车项使用 `t-swipe-cell` 承载左滑删除。
- 当前工程开启了 `skylineRenderEnable`。
- `t-swipe-cell` 在当前 Skyline 运行链路下表现不稳定，导致开发者工具鼠标拖拽可见、真机触摸左滑无效。

## Scope

### In Scope

- 移除购物车场景对 `t-swipe-cell` 的依赖。
- 以页面内自管手势替换滑删交互。
- 保持现有删除按钮视觉风格与位置语义。
- 保持单项勾选、全选、汇总金额、去结算逻辑不变。
- 为滑删手势增加可回归的纯函数测试与结构测试。

### Out Of Scope

- 改购物车的视觉风格。
- 改删除文案或删除确认流程。
- 改结算逻辑、登录门禁、购物车存储结构。
- 改 Tab、角色切换或其他顾客端页面。

## Product Decisions

### 交互替换策略

保留“左滑显示删除”这一用户习惯，但不再依赖第三方组件内建手势。购物车项自身接管 `touchstart / touchmove / touchend / touchcancel`，按水平位移驱动内容层平移，露出右侧删除操作区。

### 稳定性策略

- 仅允许一项处于展开态。
- 垂直滚动优先，只有当水平位移超过最小阈值且大于垂直位移时才进入滑动状态。
- 结束拖拽后按阈值决定“完全展开”或“回弹关闭”。
- 点击已展开项内容区时先关闭，不误触其他操作。

## Data Design

### 购物车显示态扩展

`customer-cart-scene` 的显示数据增加以下临时字段：

- `swipeOffset`: 当前项已左移的像素值。
- `swipeOpen`: 当前项是否处于展开态。

这些字段只存在于页面显示层，不写回购物车持久化存储。

### 手势状态

新增一个纯工具模块维护手势判定规则：

- 方向判定
- 位移范围裁剪
- 展开/关闭阈值判断

这样既能让组件逻辑更轻，也能用 Node 测试直接覆盖。

## Module Design

### `customer-cart-swipe.ts`

职责：

- 判断当前手势是水平还是垂直。
- 根据起点位移和本次拖动量计算下一帧偏移。
- 根据阈值决定收手后应展开还是关闭。

### `customer-cart-scene.ts`

职责调整：

- 管理单项展开状态。
- 响应触摸事件并同步 `items` 的显示偏移。
- 保持现有勾选、全选、删除、去结算逻辑。

### `customer-cart-scene.wxml / wxss`

职责调整：

- 用“操作层 + 内容层”替代 `t-swipe-cell`。
- 保持现有卡片结构和删除按钮外观。

## Error Handling

- 触摸数据缺失时直接忽略，不抛异常。
- 删除项不存在时沿用现有 `removeCartItem` 行为，安全返回。
- 购物车重新同步时清空临时滑动状态，避免脏 UI 残留。

## Testing Strategy

### Unit Tests

- 新增 `customer-cart-swipe.test.ts`
  - 水平/垂直方向判定
  - 位移裁剪
  - 收手后展开/关闭决策

### Structural Tests

- 更新 `customer-cart-scene.test.ts`
  - 不再依赖 `t-swipe-cell`
  - 改为断言自管触摸事件与删除操作层存在

### Manual Verification

使用微信开发者工具 `2.01.2510280` 与手机扫码真机验证：

1. 左滑任一购物车项，可稳定露出删除按钮。
2. 上下滚动购物车时，不会被误判成左滑。
3. 点击删除，仅删除对应项。
4. 勾选、全选、合计金额、去结算行为与修复前一致。
